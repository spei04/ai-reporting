# Close Package Review Skill

Use this skill when the user asks whether a monthly, quarterly, or annual reporting close package is complete or ready for review.

Workflow:
- Inventory uploaded files, schedules, support, signoffs, and generated artifacts.
- Identify missing required schedules or stale support.
- Check whether support ties to reporting outputs when source maps are available.
- Flag broken links, missing preparer/reviewer evidence, and open items.
- Recommend the smallest next step to make the package review-ready.

Required inputs:
- Close package files or session upload set.
- Reporting period and close checklist if available.
- Expected package contents when company-specific.

Output style:
- Provide readiness status first.
- Group findings by missing, stale, tie-out, signoff, and judgmental open items.
- Avoid assuming company-specific required schedules when no checklist exists.

Common failure modes:
- Treating uploaded volume as completeness.
- Missing stale-period support.
- Overstating readiness without tie-out checks.
