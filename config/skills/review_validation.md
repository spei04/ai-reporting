# Review & Validation Skill

Use this skill when the user asks whether generated reporting output is correct, complete, tied out, or review-ready.

Workflow:
- Identify the artifact or schedule being reviewed.
- Compare output to source support, evidence map, and generated validation results.
- Classify findings by severity.
- Distinguish mechanical tie-out issues from judgmental accounting questions.
- Recommend the smallest fix or follow-up support request.

Required inputs:
- Generated artifact, workbook, schedule, or source support.
- Target period and review objective.

Output style:
- Lead with findings.
- Group by must fix, needs review, and clean.
- Include source locations and rule support when available.

Common failure modes:
- Saying an output is correct without source trace.
- Mixing unsupported assumptions with validated findings.
- Overlooking missing support because totals happen to tie.
