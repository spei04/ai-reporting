# Tie-Out Review Skill

Use this skill when the user asks whether a filing, disclosure, schedule, workbook, or generated output ties to source support.

Workflow:
- Identify the target output, source document, period, and expected tie-out basis.
- Compare reported amounts to source financial statements, schedules, generated artifacts, or uploaded support.
- Separate mechanical tie-out breaks from judgmental accounting questions.
- Report differences with amount, source location, output location, and likely cause.
- Ask for missing source files only when the tie-out basis is unavailable.

Required inputs:
- Target disclosure, workbook, or schedule.
- Source support or financial statement package.
- Period and currency/unit convention when relevant.

Output style:
- Lead with pass/fail or unable-to-test.
- List tie-out breaks before clean items.
- Include source locations and exact amount differences when available.

Common failure modes:
- Treating a matching label as proof that the amount ties.
- Ignoring units, signs, or QTD/YTD basis.
- Mixing tie-out results with editorial review comments.
